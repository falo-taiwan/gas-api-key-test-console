API Playground（執行頁）

這裡是用來測試 API Key 是否可用，以及 Gemini / Kimi / MiniMax 是否能正常串接。同時也可以測試登入驗證、後端即時設定覆蓋與 debug 回傳。

這章在學什麼：把前端輸入、後端 Sheet 設定與 GAS 回傳連成完整流程。

實務上拿來做什麼：給學員、客戶或團隊成員自己測 key、測登入、測輸出限制。

哪裡容易踩雷：忘了填 GAS URL、欄位名稱不一致，或不清楚目前限制值來自前端還是後端。

這裡預設留空。GitHub 公開版不內建真實 exec，避免直接暴露後端入口；若要測試，請手動貼上你的 Web App URL，或向管理者索取受控測試網址。
教學版預設帳號可用 admin 與 user。
若要帶入管理者目前密碼，請按下方「填入管理者示範值」並輸入驗證碼；後端會讀取 users sheet 中 admin 的目前密碼。
金鑰來源
如果留空，後端會讀取 Google Sheet 的 runtime_config.system_prompt。
如果留空，後端會使用 runtime_config.output_char_limit；若輸入 0，代表不限制字數。

管理者示範值會先帶入 admin、模型、提示詞與示範問題，再要求你輸入第二組驗證碼；驗證成功後，後端會讀取 users sheet 中 admin 的目前密碼並回填。一般使用者示範值則會帶入 user / 123。注意：只有在 Use Server Key 模式下，user 才會進入受限教學回覆；若切到 Use My API Key，則不受角色限制。

安全備註：這份公開版已移除內建真實 exec。一般情況下，真正較高風險的是可直接呼叫後端的 GAS Web App URL；因此若要對外分享，建議改用受控發放、手動貼上，或另建專用測試端點。
